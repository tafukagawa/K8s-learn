03 プラットフォーム分離

ElectronとWebで同じReactコードを使いながら、データ取得層だけを切り替えている。Viteの resolve.alias を利用してビルド時にファイルをスワップする。

vite.web.config.ts 
// ipc への import を ipc.web.ts に差し替え
resolve: {
  alias: [
    {
      find: /.*\/shared\/ipc$/,
      replacement: 'src/shared/ipc.web.ts',
    },
  ],
},
ファイル用途データソース
src/shared/ipc.ts Electron版 API クライアント window.api（IPC経由でSQLite）
src/shared/ipc.web.ts Web版 API クライアント JSONファイル + localStorage
コンポーネント側は import { api } from '../../shared/ipc' とするだけ。ElectronかWebかを意識しない。

04 データ層

コンテンツファイル構造（Web版）

categories/ ├── k8s/ │ ├── meta.json { "name": "Kubernetes" } │ └── sections/ │ └── 01-pod-management/ │ ├── meta.json { "title": "Pod管理" } │ ├── commands.json コマンド一覧 │ └── knowledge.json ナレッジ一覧 ├── docker/ ├── helm/ ├── argocd/ └── kustomize/

07 UIデザイン

テーマシステム

src/shared/theme.ts で MUI の createTheme() を使って dark/light テーマを定義。App.tsx の state でトグル。

要素手法
スタイリング MUI の sx prop。CSS-in-JSで型安全なスタイル記述
透過色 MUI の alpha() ヘルパー関数でテーマカラーに透過度を付与
アイコン（ナビ） @mui/icons-material（MUI Icons）を使用
アイコン（UI） @primer/octicons-react（GitHub製）を使用
ダークモード デフォルトdark。ヘッダーのトグルボタンで切り替え。localStorageには未永続化
ドラッグハンドル DragIndicatorIcon（MUI）をホバー時のみ表示（opacity: 0→1）
通知 MUI Snackbar + Alert。ポーリング完了・エラー通知に使用

ドラッグ＆ドロップ（dnd-kit）

カテゴリの並び替えに使用。DndContext + SortableContext でラップし、useSortable フックを持つ SortableCategoryRow コンポーネントに分離。

Sidebar.tsx — 構造
DndContext collisionDetection={closestCenter} onDragEnd={handleDragEnd}
  SortableContext items={catIds} strategy={verticalListSortingStrategy}
    SortableCategoryRow ← useSortable({ id: cat.id })
      DragIndicatorIcon  ← onClick: stopPropagation
      ListItemButton     ← onClick: toggleCat

08 カスタムフック

useWorkflowPolling()
GitHub Actions ワークフローの完了をポーリング監視するフック。
src/shared/hooks/useWorkflowPolling.ts

status: idle → polling → done | error
startPolling(token, dispatchedAt): ポーリング開始
dismiss(): 通知を閉じる

KnowledgeList レベルで保持することで、GenerateFromUrlDialog を閉じてもポーリングが継続する。最大10分（10秒×60回）。

09 GitHub Actions ワークフロー

ファイルトリガー内容
deploy.yml main へのpush build:web → GitHub Pages へデプロイ
generate-from-url.yml workflow_dispatch（アプリUIから） URL指定→GitHub Models APIでナレッジ生成→categories/ にコミット
generate-content.yml 手動 / スケジュール sectionUrls.ts に定義されたURL群からまとめてコンテンツ生成
generate-from-url は permissions: models: read が必要（GitHub Models API アクセスのため）。
